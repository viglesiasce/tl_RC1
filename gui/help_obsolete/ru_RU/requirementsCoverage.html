Обзор

Эта функция показывает степень покрытия требований тестовыми примерами. Для просмотра требований пользуйтесь ссылкой "Редактировать требования" на рабочем столе.

Группы требований

Требования собраны в группы, связанные с продуктом. TestLink не поддерживает версионность для требований и их групп.

Общее количество требований служит для оценки покрытия требований тестами, поэтому не все требования, предъявляемые к программному продукту, целесообразно вносить в TestLink. Значение поля "Всего требований" определяет количество требований, используемых для построение отчетов.

Например, некая программа включает 200 требований, но только 50 тестов внесены в TestLink. Тестовое покрытие в этом случае составляет 25% (если все требования - тестируемы).

Требования

Нажмите на название уже созданной группы требований. Для этой группы вы может создавать, редактировать, удалять или импортировать требования. Каждое требование включает название, содержание и статус. Стутус определяет, является ли данное требование тестируемым или нет. Нетестируемые требования не подсчитываются при формировании отчетов.

На странице "Редактор требований" вы можете создавать тесты привязанные к выбранным требованиям. Эти тесты будут созданы для компонента под название "Component/Category/Test Cases generated from Requirements" в категории, имя которой совпадает с именем группы требований.
